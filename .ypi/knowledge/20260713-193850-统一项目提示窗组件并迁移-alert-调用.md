# 统一项目提示窗组件并迁移 alert 调用

## Summary
新增应用级统一提示系统，覆盖 notice、confirm、prompt 和 toast；迁移生产代码中的 14 处原生 confirm/prompt 调用，保持业务条件、文案与返回值语义。任务已完成人工确认并归档。

## Reusable knowledge
- 使用单一 `AppPromptProvider` + Context + portal，提示请求按 FIFO 串行处理。
- 每个请求必须 exactly once settlement；Provider 卸载时同时结算 active 和 queued 请求，避免 Promise 悬挂。
- `confirm` 取消返回 `false`；`prompt` 取消返回 `null`，空字符串表示清除内容，不能使用 falsy 判断混淆两者。
- Dialog 需要处理焦点恢复、Tab focus trap、Escape、IME-safe Enter、scroll lock、背景 `inert`、响应式和 reduced-motion。
- Toast 不抢焦点，限制最多三条，使用 live region，并清理自动关闭 timer。
- 迁移异步确认时保留原业务顺序：取消必须提前返回，确认操作只能执行一次。
- 验证至少执行：原生调用 `rg` 扫描、`git diff --check`、`npm run lint`、`node_modules/.bin/tsc --noEmit`，并进行桌面/移动端人工验收。
