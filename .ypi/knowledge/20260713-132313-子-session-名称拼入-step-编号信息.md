# Summary

YPI Studio 派发子代理产生的 child session 侧栏标题，原先主要用任务名/子任务标题，缺少任务面板中的 step 编号信息。本任务将稳定 **`subtask.id`** 拼入子 session 名称，并统一侧栏展示与新 child `session_info` 落盘命名。

## 标题规则

| 场景 | 格式 |
| --- | --- |
| 有 subtask | `{subtaskId} · {subtaskTitle}` |
| 仅有 id | `subtaskId` |
| 无 subtask | `{member} · {taskTitle}`（不造伪编号） |
| 50 字截断 | **id > title > member** |

# Reusable knowledge

1. **Step 编号口径**：用稳定 `subtask.id`，不用 plan 的 1-based index；与任务面板 `{id} · {title}` 一致。
2. **单一 helper**：`lib/session-title.ts` 的 `studioChildSessionTitle()` 同时服务 `displayTitleForSession` 与 `studioChildSessionInfoName`，避免 runner/UI 规则分叉。
3. **存量兼容**：不改 `studioChild` header schema、不回写历史 JSONL；`projectStudioChildDisplay` 读时投影 `subtaskId`/`subtaskTitle`/`taskTitle`。
4. **投影缓存身份**：cache key 必须包含 `cwd + taskId + subtaskId + runId`，否则同任务多 child 会串标题/runSummary。
5. **信息分工**：主标题优先 step id/title；member/status/run short id 放 badge、detail、tooltip。
6. **UI 门禁**：侧栏可见信息结构变化需 HTML 原型 + 用户审批后再实现。
7. **验证**：`npm run test:session-title`、`test:studio-sdk-runner`、lint、tsc。
