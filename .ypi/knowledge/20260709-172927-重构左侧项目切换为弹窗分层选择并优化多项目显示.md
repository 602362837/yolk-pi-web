# Summary

已完成左侧项目切换重构：移除 sidebar 内旧 dropdown，改为顶部切换按钮 + 项目/空间分层弹窗，并补齐多项目展示、空状态 onboarding、添加项目流程迁移、WorkTree 右键入口与可访问性/响应式优化。

# Reusable knowledge

- 优先纯前端重构，不改变 Project Registry、会话 JSONL、后端 API 与 WorkTree 语义。
- 项目切换适合采用“左侧项目、右侧空间”的分层弹窗，而不是在窄 sidebar 内继续堆 dropdown。
- 大量项目场景需要固定搜索框、左右独立滚动、长文本省略与 tooltip。
- 零项目环境必须在同一弹窗内提供 onboarding，并复用已有默认目录/文件夹/路径/Git clone 注册流程。
- WorkTree 相关右键入口不能因弹窗重构而丢失；checker 已确认这是一个实际 blocker。
- 验证基线至少包括 `npm run lint`、`node_modules/.bin/tsc --noEmit`，以及 50+ 项目、missing space、空状态、键盘焦点/Esc/窄窗口等手工检查。
