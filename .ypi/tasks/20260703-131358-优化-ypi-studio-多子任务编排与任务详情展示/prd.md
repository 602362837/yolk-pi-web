# prd

## 目标

让 YPI Studio 的多子任务能力从“只知道一组 subtask”升级为“能表达、展示、刷新、执行串并行关系”的任务详情体验。

## 用户可见行为

- 架构师拆解时，每个子任务都有执行关系：串行、并行组、依赖项、可并发说明。
- 任务详情流程路线展示：串行阶段按顺序展示，并行阶段合并为并行组，标明组内任务可同时执行。
- 实现 tab 对多子任务使用二级 tab：默认选中当前 running/ready/最近更新子任务，只渲染当前选中内容。
- 未完成子任务显示占位状态，不再把已经带 `.md` 的文件名再次追加 `.md`。
- 工作室抽屉后台刷新静默更新，不插入额外刷新提示行；用户当前阅读位置和选中 tab 保持稳定。

## 非目标

- 不绕过用户审批门禁。
- 不要求一次实现真正的多 worker 并行执行；本次优先结构化数据与 UI 呈现，编排可先保持主会话 claim 一个 ready subtask 的安全模式。
