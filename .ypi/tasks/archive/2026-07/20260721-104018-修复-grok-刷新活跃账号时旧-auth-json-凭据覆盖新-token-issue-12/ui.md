# UI：无需原型

## 门禁结论

**不触发 UI 原型门禁。**

本任务只修复 Grok OAuth saved-account 与 `auth.json` Active 镜像之间的服务端持久化和并发一致性，不新增或修改：

- 页面或组件；
- 前端功能；
- 现有交互；
- 审批/确认体验；
- 用户可见文案或信息结构。

因此不派发 UI 设计员，不制作 HTML 原型，也不需要 UI 原型审批。

## 范围变化门禁

若实现中提出新增“凭据冲突”“自动修复完成”“请重新登录”等可见提示，或增加人工恢复入口，必须停止当前实现、重新判定为 UI 变更，并先由 UI 设计员基于现有项目交付 HTML 原型给用户审批。
