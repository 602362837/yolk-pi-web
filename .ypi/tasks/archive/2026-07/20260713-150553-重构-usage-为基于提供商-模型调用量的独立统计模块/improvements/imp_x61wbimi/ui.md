# UI 设计说明

对于改进项 **IMP-003**（用量展示优化计划），我们设计了相关的交互原型与界面改进，主要解决以下问题：
1. **用量明细精度**：在明细表格和面板中，所有 Token 数均默认显示完整数字，不再使用 `M/K` 进行精度隐藏，并增加千分位格式。
2. **悬浮详细提示 (Tooltip)**：对 Token 数量、预估费用、缓存命中率等关键指标设计了悬浮提示（Tooltip）以展示完整值、计算口径和物理拆分公式。
3. **缓存命中率 (Cache Hit Rate)**：细化模型明细的缓存命中率计算逻辑，计算公式为：`Cache Read / (Input + Cache Read)`。如果分母为 0 则显示 `—`。
4. **每日多模型堆叠柱状图 (Stacked Bar Chart)**：
   - 将原有的单色费用柱状图，升级为按模型分配稳定颜色的堆叠柱状图。
   - 支持“使用量”与“费用”两种模式的实时切换（默认使用量模式）。
   - 在“费用”模式下，如果模型没有配置费用，柱状图中将显示“无费用数据”状态，而不影响“使用量”模式的统计展示。
5. **多状态适配**：对无数据、加载中、服务端错误、部分事件损坏、隔离等状态进行了视觉覆盖。

## 交互设计与状态对照

| 场景 | 展示组件 | 用户操作 | 反馈 |
| --- | --- | --- | --- |
| 默认浏览状态 | 明细表格 & 堆叠图 | 鼠标悬浮在 Token 数量上 | 悬浮卡片显示物理拆分细节：`总 Token = Input + Output + Cache Read + Cache Write`，包含推理 Token (Reasoning) |
| 浏览堆叠柱状图 | 堆叠图图表卡片 | 点击 `使用量 / 费用` 切换按钮 | 柱状图根据模型使用量/预估费用，切换为不同颜色 and 占比长度的堆叠块展示 |
| 无配置费用的模型 | 堆叠柱状图费用模式 | 切换到 `费用` 模式，且当天仅有无费用的本地模型 | 柱状图中渲染为灰色斑驳样式，并显示“无费用数据”文字提示 |
| 缓存读取为 0 | 缓存命中率列 | 无 | 显示 `—` |
| 查看模型细分详情 | 明细表格行 | 点击任意模型行 (如 `gpt-4o`) | 右侧滑出该模型的 Drawer，以更清晰的可访问性等宽字体排版该模型的来源分布与详细指标 |

## 实施注意事项 (Implementation Notes)

1. **缓存命中率计算**：
   - 必须使用公式 `totals.cacheRead / (totals.input + totals.cacheRead)`，且需做分母防御。
   - 避免直接将 `cacheRead / totals.totalTokens` 作为命中率。
2. **图表配色稳定性**：
   - 在前端根据 `provider::model` 的字符串 Hash 或固定索引，为其分配预设的调色板（Palette）颜色，确保同一个模型在同一天的图表中拥有统一的视觉指示色。
3. **可访问性与响应式**：
   - 窄屏模式下，表格应隐藏非核心列（`Input`、`Output`、`Cache R/W`、`缓存命中率`、`占比` 等），并在表格下方提示用户点击模型行滑出抽屉查看完整细节。
   - 账本所有的数值均使用 `font-variant-numeric: tabular-nums` 样式，保证数字对齐和比对体验。
   - 页面与抽屉支持 `ESC` 快捷键关闭。

## 验收清单与检查项 (UI Checks)

- [ ] 所有的 Token 数（如 Input, Output, Cache Read 等）在默认显示时均为千分位格式的完整数字。
- [ ] 悬浮在表格 Token 及命中率上时，可以看到详细公式展示。
- [ ] 点击图表顶部的 `使用量 / 费用` 能成功切换柱状图的占比模式。
- [ ] 若当天仅有 `qwen2.5-coder-7b` 等零费用本地模型，在 `费用` 堆叠柱状图模式中会显示“无费用数据”状态。
- [ ] 当 `Input` 与 `Cache Read` 均为 0 时，表格对应的模型“缓存命中率”渲染为 `—`。
- [ ] 窄屏（手机宽度）下表格无水平溢出滚动条，且点击模型能正常弹出侧栏抽屉展示完整信息。
