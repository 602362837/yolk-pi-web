# ui

## 交互核心：分享码绑定，避免错 session

采用“插件生成分享码 + 目标 chat 填码绑定”的双端确认模型。

### Chrome 插件 popup

状态一：未连接 ypi
- 标题：YPI Browser Share
- 说明：请先启动 ypi web。
- 按钮：检查连接

状态二：已连接但未分享
- 显示当前 tab title / origin。
- 主按钮：分享当前页
- 权限选择：只读（默认）/ 允许操作

状态三：待绑定
- 显示大号分享码：`K7Q4-9P`
- 显示倒计时：5:00
- 说明：在目标 ypi 聊天中输入此码完成绑定。
- 按钮：取消分享 / 重新生成

状态四：已绑定
- 显示：已绑定到 ypi session
- 显示权限：只读 / 可操作
- 按钮：停止分享 / 降级为只读

### ypi ChatInput 附近

新增 Browser Share 入口，建议放在 ChatInput 工具栏，与图片上传/工具 preset 同层：

- 未绑定：按钮“绑定浏览器分享”
- 点击后弹出小面板：输入分享码、绑定按钮、说明“分享只会绑定到当前聊天”
- 已绑定：显示 pill “Chrome 已共享：<title>”
- pill 点击打开详情：URL、权限、最后更新时间、解绑、提升权限请求

### 会话内展示

在 ChatWindow 或浮动小卡片显示当前 session 的 Browser Share 状态：

- 连接中：绿色点 + 页面标题
- 只读：灰/蓝标签
- 等待确认操作：黄色提示 + 允许/拒绝
- 断连/过期：红/灰提示 + 重新绑定

### 操作确认

agent 请求操作时，UI 展示：

- 操作类型：点击/输入/滚动/导航
- 目标元素摘要：按钮文本、label、role、附近文本
- 风险提示：输入文本是否敏感、是否提交表单、是否离开当前域名
- 操作按钮：允许一次 / 拒绝 / 本 session 始终允许低风险操作

### 防误分享细节

- 分享码只在插件端显示，不自动绑定最近 session。
- ypi 绑定面板必须明确显示“当前 chat/session”。
- 一个分享码只能绑定一次，绑定后立即失效。
- 已绑定 session id 在插件端以短标签展示，用户可停止分享。
