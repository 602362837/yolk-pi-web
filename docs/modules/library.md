# Library Module Map

Shared logic lives under `lib/`. Prefer adding behavior here when it is used by multiple API routes, hooks, or components.

| File | Purpose |
| --- | --- |
| `lib/project-registry-types.ts` | Shared Project Registry schema and wire types for projects, spaces, worktree metadata, and metadata patch payloads. `PiWebProjectSpaceRecord` includes optional `sortOrder?: number` for persisted user-defined non-main space ordering (smaller values sort earlier; `main` ignores this field). WorkTree space metadata may include `branch`, `repoRoot`, `mainWorktreePath`, `mainWorktreeBranch`, optional creation-time `baseRef`, and `discoveredAt`. |
| `lib/project-registry.ts` | Project Registry persistence under `~/.pi/agent/pi-web-projects.json`: canonical path/realpath handling, atomic read/write, project registration with main-space creation, duplicate active-project detection by `pathKey`, project/space metadata patch helpers, `computeNextSortOrder(project)` helper for new-space sort-order assignment using interval stepping (max + 1024), Git worktree space sync from `git worktree list --porcelain` (`syncProjectWorktreeSpaces`) with new-space `sortOrder` auto-assignment and existing-space `sortOrder` preservation, creation-time WorkTree `baseRef` preservation when available, unified worktree space archive helper (`archiveWorktreeSpacesByPaths`) with multi-path-alias matching (`pathKey` primary, `displayPath`/`realPath` fallback), lightweight passive missing-only sync (`syncMissingWorktreeSpaces`) for detecting filesystem-removed worktree spaces, legacy `markWorktreeSpaceArchivedByPath` (now delegates to `archiveWorktreeSpacesByPaths`), and `reorderProjectSpaces(projectId, orderedSpaceIds)` for batch-reordering non-main spaces with validation (rejects main/unknown/archived/duplicate/cross-project ids, appends non-payload active spaces to the end, writes interval-based `sortOrder` values at 1024 steps). |
| `lib/project-display.ts` | Project/space display-name helpers (`displayProjectName`, `displaySpaceName`), `activeProjectSpaces()` sorting (main always first, non-main by `sortOrder` ascending with `createdAt` fallback for legacy spaces — `pinned` no longer affects space ordering), `sortProjectsForSidebar()` (pinned-first, then lastOpenedAt/updatedAt, then name), and `worktreeInfoFromSpace()` / `shortenCwd()` helpers. |
| `lib/project-session-index.ts` | Best-effort `~/.pi/agent/pi-web-session-index.json` maintenance for new/forked sessions linked to project spaces; session headers remain the source of truth. |
| `lib/session-project-link.ts` | Helpers for reading/writing optional `projectId`/`spaceId` on session headers, deriving `legacyUnassigned`, and canonical path matching for legacy exact-cwd display. |
| `lib/rpc-manager.ts` | `AgentSessionWrapper`, global registry, `startRpcSession()`, cwd-scoped session cleanup, lifecycle handling, built-in YPI Studio extension factory injection for web-created AgentSessions, Studio child terminal continuation prompts, idle-timeout extension while Studio children are active, and ChatGPT account failover retry hook wiring. Main sessions use `createWebAgentSessionServices` + `createAgentSessionFromServices` (provider-aware `ModelRuntime`). Async `reloadRpcAuthState()` offline-refreshes every live wrapper's runtime, replaces same provider/id model descriptors without `setModel()` / settings defaults, then cleans provider session resources; Activate/login/logout callers must await it. Chat `set_model` is session-scoped (plan A): it resolves models via `modelRuntime.getModel` and runs SDK `setModel` under `withSessionScopedSettingsDefaults` so runtime model + JSONL `model_change` still apply while `settings.json` `defaultProvider`/`defaultModel` (and incidental default thinking from re-clamp) are not rewritten. Explicit Settings/CLI default-model writers remain untouched. **Server cold-start defaults (plan B · MODEL-PIN-CS-03):** `applyWebSessionColdStartDefaults(wrapper, sessionFile)` runs after wrapper creation with priority **session recoverable model > yolk specific + thinking > SDK/settings**. Recoverable model (path `model_change` entry + runtime available) is restored session-scoped via `withSessionScopedSettingsDefaults`. Otherwise `yolk.defaultModel` (when `mode === "specific"`) is applied as the initial runtime model + thinking; `piDefault` keeps SDK/settings default. Studio child sessions (header `studioChild`) are skipped so user yolk does not pollute subagent policy. All yolk applies are session-scoped (MODEL-PIN-3: never writes `settings.json`). Apply failures are logged and do not block session creation; the client-side `set_model` before prompt remains the primary model guarantee. |
| `lib/session-model-pin.ts` | Pure Chat model pin/display helpers (`normalizeSessionModelRef`, `resolveDesiredSessionModel`, `resolveChatDisplayModel`, `shouldPinSessionModel`, `clampThinkingLevelToSupported`) plus `withSessionScopedSettingsDefaults` for MODEL-PIN-3: Chat `set_model` keeps runtime + JSONL `model_change` while suppressing SDK writes to `settings.json` global defaultProvider/defaultModel (and incidental default thinking from setModel re-clamp). Shared SettingsManager instances are refcounted so concurrent/nested scopes restore originals correctly. Thinking clamp prefers keep-current → medium → auto → first supported. **Cold-start helpers:** `shouldPinSessionModel` accepts optional `{ liveConfirmed }` — when `false` (cold start, idle destroy) any valid desired model forces pin even if equal to stale `lastPinned`. `resolveYolkColdStartModel(yolkConfig)` parses yolk specific model + thinking; returns `null` for `piDefault`. `resolveColdStartModelPreference({ recoverable, yolk })` encodes the pure priority: session recoverable > yolk specific > SDK/settings. File-header invariants: lastPinned/confirmed must not come from path context.model; Chat set_model is session-scoped; yolk cold-start apply must also use `withSessionScopedSettingsDefaults`. |
| `lib/agent-session-bootstrap.ts` | Shared API helper for validating/canonicalizing cwd, creating configured empty AgentSessions, registering allowed roots, and applying initial tool/model/thinking selections before a prompt is sent. |
| `lib/session-metadata-scanner.ts` | Project-owned bounded streaming JSONL inventory scanner. Enumerates active/archive `*.jsonl` with fixed concurrency (default 8), reads by chunk (not full-file `readFile`, not whole-line `JSON.parse`, not `SessionManager.getEntries()`), and returns `LightweightSessionMetadata` only: path/id/cwd/name/parentSessionPath/created/modified/messageCount/bounded firstMessage (default 100 chars). Skips message/tool/compaction/custom content after capture budgets; never builds `allMessages` / `allMessagesText`. Exports `scanSessionMetadata`, `scanSessionMetadataFromChunks`, `scanSessionFiles`, `scanSessionInventory`. Malformed/orphan files return null and are omitted. |
| `lib/session-reader.ts` | Parse `.jsonl` session files, resolve session paths, and map lightweight inventory into API `SessionInfo` wire summaries. Active inventory, allowed-roots cwd discovery, and delete-by-cwd call `scanSessionInventory()` instead of SDK `SessionManager.listAll()`. Git/worktree metadata is opt-in; YPI Studio child audit sessions are hidden by default unless `includeStudioChildren: true` (Usage/audit). Studio child summaries set `legacyUnassigned=false` without changing normal sidebar defaults; UI callers may opt in to `studioChildDisplay` projection from Studio task metadata so child titles prefer stable `subtaskId` + title (or member/task fallbacks) without adding task I/O to usage scans. `projectStudioChildDisplay()` is cached by `cwd:taskId:subtaskId:runId` so same-task children do not share the wrong title/summary, and historical children get the new title via read-time projection without JSONL migration. Also prunes/deletes sessions for removed WorkTree cwd paths and reads model/default config. Archive helpers: `getSessionsArchiveDir()`, `archiveSessionFile()`, `unarchiveSessionFile()`, `scanArchivedCwds()` (header-only cwd counts), `listArchivedSessionsForCwd()` / `listAllArchivedSessions()` (lightweight scanner over `sessions-archive`), `listAllArchivedSessionMetadata()` (header-only metadata for Usage session sets), `resolveArchivedSessionPath()`. Single-session detail still uses `SessionManager.open` / `getEntries` when needed. |
| `lib/session-header-metadata.ts` | Pure parser for session header metadata shared by session listing/tests, including optional Project Registry linkage and YPI Studio `studioChild` audit metadata extraction. |
| `lib/session-title.ts` | Pure helpers for deriving display titles from sessions and first user messages, including the pending empty-session label used by Browser Share precreated sessions. Studio child sessions use `studioChildSessionTitle()` (shared by sidebar `displayTitleForSession()` and new SDK `session_info` names): stable `subtaskId · subtaskTitle` first, then id-only, then `member · taskTitle` (no fake step numbers), then member-prefixed run/task fallbacks, all under the 50-char budget with id/title priority over member. Ordinary session `name -> firstMessage -> pending/id` fallback is unchanged. **SCI:** `sessionTitleSeedFromUserMessage` and `displayTitleForSession` (firstMessage path) call `stripYpiStudioInjections` so historical dirty user text does not seed sidebar titles; stored session metadata is never rewritten. |
| `lib/types.ts` | Shared TypeScript types for messages, sessions, Studio child-session header metadata, Chat top-bar Studio child inventory wire types (`StudioChildPanelStatus`, `StudioChildSessionListItem`, `StudioChildSessionListResponse` with fixed `limits.terminal: 20`), Git/worktree metadata including optional `baseRef`, Git status/graph/commit/diff wire payloads, and API payloads. | `lib/antigravity-usage-ring.ts` | Pure Antigravity top-bar ring projection (no React/network/credentials). Groups safe windows via `antigravity-quota-groups`, then builds **independent** single-layer `ringSlots` for priority groups (Flash \| Opus side-by-side, `mode: dual-independent` / `single`). Non-priority-only → detail-only (`ringUnit=null`, fixed “多模型/详情”). Aggregate projection may emit `ringUnits` (multi independent) with risk from worst group tone; never total/avg percent. | `lib/oauth-accounts.ts` | Persist, import raw/converted credential JSON, sanitize, sync, label, activate, quota/reset-credit cache metadata, and soft-delete saved OAuth accounts without exposing tokens. Supports `openai-codex`, `grok-cli`, `kiro`, and `google-antigravity` through provider adapters; delegates credential validation, account-id derivation, display hints, and import support to `lib/oauth-account-providers.ts`. Saved-account `accountId` is an opaque storage id (including legacy v1 ids); credential `accountId` remains the real provider-native id. New imports allocate one storage id per item, including duplicate credentials, and CPA conversion warnings are returned without credential material. Active-account metadata uses a compare-and-set mirror to `auth.json`. Credential files are `0600`, directories `0700`, with atomic tmp+rename writes. Deleted accounts move to `deleted/` subdirectory; deletion of active accounts returns 409 unless a replacement or explicit disconnect is specified. Antigravity Activate shares `withAntigravityProviderLock()`. |
| `lib/oauth-account-converters.ts` | Shared OAuth account import mode registry, raw credential validation, and CPA/SUB2API-to-raw conversion used by the UI and account import API; SUB2API exports may convert to multiple raw credentials. Exports `KIRO_PROVIDER_ID`, `ANTIGRAVITY_PROVIDER_ID`, `getOAuthAccountAdapter()`, `isSupportedOAuthAccountProvider()`, and `maskAccountId()`. |

| `lib/web-auth-config-value.ts` | API-key config-value resolver for CredentialStore `read()` (literal, `$ENV`/`${ENV}`, escapes, leading `!command`). Shared by the Web store so Active API keys keep historical semantics. |

## Key Registry Functions

### `archiveWorktreeSpacesByPaths(paths, options?)`

Unified WorkTree space archiving helper. For each input path, derives canonical `pathKey` / `displayPath` / `realPath` via `canonicalizeProjectPath()` and matches against worktree spaces (`space.kind === "worktree"`) using `pathKey` as the primary key with `displayPath` and `realPath` as fallbacks. Matched spaces are soft-archived (`archived: true, missing: true`) with additive audit metadata (`metadata.archivedReason`, `metadata.archivedAt`, `metadata.lastKnownPath`) — never hard-deleted from the registry.

- `paths`: One or more path aliases (e.g., `cwd`, `status.cwd`, `status.worktree.repoRoot`).
- `options.reason`: Audit label stored in metadata (default `"api_archive"`).
- `options.missing`: Whether to set `missing: true` (default `true`).
- Returns `{ archivedSpaces, unmatchedPaths }`.

### `syncMissingWorktreeSpaces(options?)`

Lightweight passive missing-only sync for worktree spaces whose directories no longer exist. Scans non-archived worktree spaces, checks filesystem existence via `canonicalizeProjectPath()`, and soft-archives spaces with missing paths. No Git commands are executed. Internally delegates to `archiveWorktreeSpacesByPaths` for consistent matching and audit metadata.

- `options.projectId`: Optional single-project scope.
- `options.reason`: Audit label (default `"passive_missing"`).
- Returns `{ archivedSpaces, unmatchedPaths }`.

### `reorderProjectSpaces(projectId, orderedSpaceIds)`

Batch-reorder non-main spaces for a single project. Validates that `orderedSpaceIds` is a string array; rejects `main`, unknown, archived, duplicate, or cross-project ids. Active non-main spaces missing from the payload are appended to the end while preserving their relative effective order. Writes clean interval-based `sortOrder` values (1024, 2048, …) to support future local insertions. Returns the updated project and all active spaces.

- `projectId`: The project owning the spaces.
- `orderedSpaceIds`: Desired order of active non-main space ids (first → last).
- Returns `{ project: PiWebProjectRecord; spaces: PiWebProjectSpaceRecord[] }`.

### `markWorktreeSpaceArchivedByPath(worktreePath)`

Legacy compatibility wrapper. Delegates to `archiveWorktreeSpacesByPaths([worktreePath], { reason: "api_archive" })` and returns just the `archivedSpaces` array. All new callers should use `archiveWorktreeSpacesByPaths` directly for multi-alias matching and `unmatchedPaths` feedback.

## Session List Reader Performance

`lib/session-metadata-scanner.ts` owns bounded inventory extraction; `lib/session-reader.ts` owns wire mapping, Studio/project projection, WorkTree prune, and the session-list snapshot cache. The cache is bounded and short-lived (1s TTL, single-flight, small key limit), shares concurrent scans, and must be invalidated after in-process session file mutations (delete/archive/unarchive). Session JSONL remains authoritative; index data cannot be used as an exclusion list without inventory/header reconciliation.

### Inventory contract

- **Retain only:** path, id, cwd, latest `session_info` name (explicit clear → undefined), parent path, created/modified, messageCount, bounded first user text.
- **Do not retain:** `allMessages` / `allMessagesText`, full message content, tool results, compaction summaries, custom entry payloads.
- **modified priority:** last user/assistant activity timestamp, else header timestamp, else file mtime — do not sort lists by mtime alone.
- **firstMessage:** string content or `type:"text"` blocks only, with a fixed char budget; UI title still uses the existing 50-char normalization (`lib/session-title.ts`).
- **Concurrency:** fixed default (8); never unbounded `Promise.all` over all files.
- **Active + archive:** same scanner, different `rootDir`; archive-all filters active inventory by canonical cwd aliases.
- **Usage:** builds session sets from lightweight list helpers (with Studio child opt-in) then opens only needed files for assistant `usage`; must not require inventory-level full text.
- **Tests:** `npm run test:session-metadata` (compatibility/escape/chunk/malformed), `npm run test:session-list-performance` (structural no-body markers + isolated-process heap gate with `--expose-gc`).
- **Rollback:** restore inventory call sites to SDK `listAll` only as an emergency code change; do not make that the long-term default.

## Reuse Rules

- Do not duplicate JSONL parsing or tool-call normalization in UI code.
- If a route and a component need the same derived value, put it in `lib/` and import it from both sides.
- Use `canonicalizeProjectPath()` for Project Registry path comparisons so display paths, symlinks, and realpath-backed `pathKey` values stay distinct.
- Treat session header `projectId`/`spaceId` as optional; legacy sessions without both fields must continue to open and should be surfaced as unassigned rather than rewritten.
- OAuth saved-account code must use `getOAuthAccountAdapter()` from `lib/oauth-account-providers.ts` instead of checking `provider === "openai-codex"`; all provider-specific logic lives in the adapter.
- Session binding uses only opaque storage ids from the adapter store; never store or log tokens, real account ids, or credential material in JSONL headers or runtime registries.
- Grok provider code must never deep-import from `pi-grok-cli/src/*`; only the default factory export is stable. Billing, token refresh, and OAuth flows are implemented independently in `lib/grok-*` modules.
- Kiro provider code must never deep-import from `pi-kiro-provider/src/*`; only the public default factory is loaded via jiti. Quota/token/failover live in `lib/kiro-*` modules and never accept credential-supplied arbitrary endpoints.
- Antigravity provider code must never deep-import from `@yofriadi/pi-antigravity-oauth/src/*`; only the public default factory is loaded via jiti after forcing callback loopback. Quota/token/failover/model-mapping live in `lib/antigravity-*` modules; never accept credential-supplied arbitrary endpoints; never depend on `pi-antigravity-rotator`.
- All Grok Web API responses carry `Cache-Control: no-store` and use the `GrokQuotaResultV1` wire projection; raw upstream payloads, base URLs, and credentials must never cross the API boundary.
- All Kiro quota responses carry `Cache-Control: no-store` and use `KiroQuotaResultV1`; never project access/refresh/clientSecret/profileArn/userInfo/email/raw bodies/paths. Failover candidates with unknown/stale quota are fail-closed.
- All Antigravity quota responses carry `Cache-Control: no-store` and use `AntigravityQuotaResultV1`; never project access/refresh/projectId/raw bodies/URLs/paths. Failover is model-aware (accepted keys only) and **not** group-aware; fail-closed on unknown/stale/unmapped/other-model-only/same-group-sibling-only quota. Top-bar priority groups render as dual-independent single-layer rings (never Flash/Opus concentric); non-priority multi-model stays detail-only without a total percent; group aggregation is conservative max(used)/min(remaining) for UI only.
- Treat session header `studioChild` as optional audit metadata. Its `parentSessionId` links to the parent chat session id, while standard `parentSession` remains the Pi JSONL parent file path; `.ypi/tasks/<task>/task.json` remains the authoritative Studio workflow/run state.

Do not estimate usage from `.ypi/.runtime/studio-subagents/*.jsonl` sidecars or CLI `--no-session` child transcripts.
