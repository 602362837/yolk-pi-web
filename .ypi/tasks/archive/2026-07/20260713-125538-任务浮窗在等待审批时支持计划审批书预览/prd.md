# PRD

## 目标与用户价值

缩短 YPI Studio 计划审批路径：当浮窗已经提示任务等待审批时，用户无需再打开右侧任务详情，即可直接审阅 `plan-review.md`。该能力只提升“看计划”的可达性，不改变“在绑定聊天中明确批准”的既有流程。

## 范围内

1. 主任务 `awaiting_approval` 的浮窗卡片显示「计划审批书」按钮。
2. 改进项 `waiting_plan_approval` 显示带改进项标识的计划审批入口。
3. 点击按钮打开模态弹窗，按需读取并用项目现有 Markdown renderer 渲染对应 `plan-review.md`。
4. 支持 loading、读取失败、缺失/空内容、长内容滚动和关闭交互。
5. Markdown 内任务相对链接沿用 task-local 安全读取/HTML preview 规则。
6. 桌面展开浮窗与移动端底部任务面板行为一致。

## 范围外

- 不在弹窗内增加“批准 / 拒绝 / 请求修改”写状态按钮。
- 不改变 approval gate/grant、revision 或 transition 规则。
- 不在浮窗 API 投影中嵌入 Markdown 正文。
- 不改右侧 Studio 任务详情现有审批 Tab。
- 不为非等待审批状态提供常驻计划入口。
- 不改变浮窗尺寸、拖拽、收纳球、排序和任务绑定规则。

## 需求与验收标准

### R1 主任务入口显隐

- 当且仅当主任务 `status === "awaiting_approval"` 时，卡片显示「计划审批书」入口。
- 主任务离开该状态后入口立即消失。
- 入口与现有详情箭头并存，不能让整张卡片变成可点击区域，也不能干扰拖拽手势。

### R2 改进项入口显隐

- 对 `task.improvements.instances[]` 中每个 `status === "waiting_plan_approval"` 的实例显示入口，文案至少能区分 `IMP-xxx`。
- 其它改进状态不显示计划审批入口。
- 多个改进项同时等待时必须可分别打开对应实例，不得默认猜测或串读另一实例。

### R3 渲染与数据来源

- 主任务读取 task root 的 `plan-review.md`；改进项通过 `improvementId` 读取实例 root 的 `plan-review.md`。
- 使用 `GET /api/studio/tasks/[taskKey]/files` 的 `mode=read`，并使用 `MarkdownBody` 渲染，不新增平行文件读取 API。
- 打开弹窗才发起读取；关闭、切换任务/session 时中止或忽略过期请求。

### R4 弹窗交互

- 弹窗显示任务标题、计划类型（主计划或改进项）、只读提示和 Markdown 正文。
- 支持右上角关闭、`Escape`、点击遮罩关闭；弹窗内部点击不关闭。
- 打开时聚焦弹窗或关闭按钮，关闭后焦点返回触发按钮；使用 `role="dialog"`、`aria-modal="true"` 和可访问标题。
- 内容区域在视口内滚动；窄屏使用接近全屏/底部安全边距布局。

### R5 状态与错误

- Loading 显示“正在读取计划审批书…”。
- 404/缺失、403/安全拒绝、网络失败显示可理解错误，并提供“重试”；不能展示空白弹窗。
- 内容为空或仍为 TBD 时显示“计划审批书尚未准备好”，不伪装成可审批材料。
- 弹窗始终提示“预览不会自动批准，仍需在绑定聊天中明确回复”。

### R6 相对链接与安全

- Markdown 相对链接不得按页面 URL 原生跳转。
- Markdown/文本文件可交给现有文件查看器；HTML 使用 task-local `mode=preview` 沙箱响应。
- scheme、绝对路径、`..`、反斜杠和越界链接继续被拒绝并显示提示。
- 改进项链接必须携带同一 `improvementId`，保持实例目录隔离。

### R7 无回归

- 非审批态卡片布局与当前一致。
- 详情箭头仍打开 Studio 任务详情；预览按钮不会触发详情打开。
- 浮窗拖拽、收纳球、本地位置持久化、任务排序和移动端面板保持可用。

## 未决问题

无阻塞产品问题。建议文案采用「计划审批书」；改进项采用「计划审批书 · IMP-001」。弹窗只读，不加入批准按钮，以避免误导或削弱现有聊天审批门禁。
