UI 交互设计与状态定义说明

1. 精确值 + M 派生并列展示规范

在全部 Usage 用户界面中，为了兼顾可复制的精确数据审计和直观的百万量级 M 单位概览，遵循如下并列显示模式：

  • 主值: 始终为精确的完整整数格式并带千分位号（例: 1,234,567 tokens）。
  • 派生值: 并列或在紧随括号中以 M 单位显示，最多保留 6 位小数，且自动去除无意义的尾部零（例: (1.234567 M)）。
  • 去尾零规则: 对于正好整数倍，显示 (2.1 M) 而非 (2.100000 M)，0 tokens 明确显示为 0 M。

2. Cache-Write 用量退役兼容规则

为遵循 prd.md 要求，从所有用户交互中抹除 Cache Write 痕迹：

  • 配置排除: 模型价格配置页表格与抽屉中不配置、不展示 Cache Write，彻底将其排除在外。
  • 计算公式: 缓存命中率更新为独立公式：Cache Read / (Input + Cache Read)。
  • 兼容API: 后端账本保持历史不可变性，新产生的数据中 cache-write 均以 0 返回（Deprecated）。

3. 智能填写与来源安全防护

  • 抓取限制: 抓取流程由后端发起，采用固定域名 allowlist 过滤器，防范 SSRF 或凭据外泄。
  • 低置信度防御: 对别名、路由代理或来源存在明显单价差异的条目（如 deepseek 聚合价格不一致），进行黄色高亮预警，并默认处于不勾选状态，要求人脑判断。
  • 非侵入性: AI 仅分析抓取的文本片段得出建议，无文件写入权，用户必须在弹框中点击“确认修改”才会计入暂存，然后再次在 Diff 确认中提交才会写入磁盘。

4. Concurrency 409 写入容错

多租户/多终端使用时，`models.json` 可能在配置期间被外部写入改变。

  • Revision ETag校验: 保存时伴随 GET 得到的版本号。如果服务端返回 409 冲突，系统将截获并弹出冲突对话框。
  • 草稿暂存保留: 本地输入的暂存数值不丢失，点击“重新读取与合并”拉取新 revision 字段后，可在保留编辑项的前提下，重新展示 Diff 进行二次提交。
