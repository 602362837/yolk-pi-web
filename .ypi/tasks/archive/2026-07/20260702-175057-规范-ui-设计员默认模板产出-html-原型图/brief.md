# Brief

## 用户目标

规范 YPI Studio 的 UI 设计员默认模板，使 UI 设计员明确必须产出 HTML 格式原型图，并在最终交付前上报给主会话/用户先审阅。

## 背景

当前 UI 设计员模板允许使用 Markdown、表格、Mermaid、ASCII 或 HTML 草图表达低保真原型，导致产出物不稳定，可能缺少可直接审阅的 HTML 原型。

## 范围

- 更新默认 UI 设计员成员模板。
- 如项目已有 `.ypi/agents/ui-designer.md` 仍是可安全更新的默认版本，同步更新。
- 更新必要文档或初始化/迁移机制，确保新工作区和旧默认模板能获得规范。

## 验收标准

- UI 设计员模板明确要求输出 HTML 原型图。
- 模板明确要求将 HTML 原型作为交付物上报，交给用户先审阅后再进入实现。
- 不覆盖用户自定义成员模板。
- 相关类型检查/静态检查通过，或说明未运行原因。
