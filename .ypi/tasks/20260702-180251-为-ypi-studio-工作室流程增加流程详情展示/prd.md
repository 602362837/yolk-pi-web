# PRD

## 用户目标

在 YPI Studio 的 Workflows tab 中，用户可以像查看任务详情一样查看某个工作室流程的详情，明确：

- 这个任务应该走哪个流程。
- 流程图/主路径有哪些节点。
- 每个节点由主会话处理还是委派给哪个成员。
- 哪些节点需要用户确认、哪些产物是必需/可选。
- 分支、阻塞、取消、归档等例外流如何走。

## 范围

- Workflows tab 支持从流程列表进入流程详情并返回。
- 流程详情展示主路径流程图、节点职责/委派、触发方式、分支与例外流、基础元数据。
- Task detail 概览页增加“当前任务流程”区块，高亮当前任务所在节点。
- 不新增 API，不改变 workflow/task 持久化 schema。

## 非目标

- 不做完整 DAG 编辑器。
- 不做 workflow URL 深链。
- 不支持在 UI 中编辑流程 JSON。
