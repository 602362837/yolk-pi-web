# PRD

## 目标与背景

上一阶段已将 Sidebar 顶部项目空间切换迁移为 `ProjectSpaceSwitchDialog` 分层弹窗，但当前操作仍分散在顶部三点菜单、WorkTree 专属右键菜单和元数据编辑弹窗中。本任务目标是在不改变 Project Registry 来源、session JSONL 与 WorkTree 后端语义的前提下，把项目/空间操作收敛到弹窗卡片上下文中，并明确“星标”和空间排序规则。

## 用户价值

- 用户在浏览项目/空间列表时，可以直接对任意 project card 或 space row 进行编辑、星标、归档、WorkTree 删除等操作，不必先切换到目标工作区。
- 星标语义统一，避免“置顶 / pinned / 星标”混用造成理解成本。
- 空间列表符合用户心智：主空间固定第一，其余空间按用户拖动顺序稳定保存，新空间默认追加到底部。

## 范围内

1. **弹窗卡片右键菜单**
   - `ProjectSpaceSwitchDialog` 左侧所有 project card 支持右键菜单。
   - 右侧所有 space row/card 支持右键菜单，包括 `main` 与 WorkTree space。
   - WorkTree 归档/删除入口并入统一 space context menu，继续复用现有确认与后端 API 语义。

2. **顶部三点菜单职责收敛**
   - Sidebar 顶部三点菜单保留。
   - 顶部菜单只处理“当前选中工作区”的快捷/全局操作。
   - 弹窗右键菜单处理“浏览态下任意 project/space”的就地操作。

3. **星标机制**
   - 数据层复用既有 `pinned: boolean` 字段。
   - UI 文案统一为“星标 / 取消星标”。
   - Project 星标继续影响项目排序。
   - Space 星标仅作为标记和快速识别，不再改变空间列表顺序。

4. **空间拖动排序**
   - `main` space 永远第一，不可拖动。
   - 非主空间可拖动排序。
   - 用户拖动顺序需要持久化，刷新/重启后保留。
   - 静默新增、Git refresh 发现或新创建的非主空间默认追加到非主空间列表底部。

5. **文档与验证计划**
   - 更新前端、库/API 相关文档入口。
   - 提供实现计划、检查清单与用户审批入口。

## 范围外

- 不重做 Sidebar 会话列表、文件树、Chat 区域或 Project Registry 顶层模型。
- 不改变 session JSONL 格式、Project-space session header link 或 WorkTree archive/delete 后端语义。
- 不在本轮加入项目列表拖动排序；项目列表仍按“星标优先 + 最近打开时间 + 名称”。
- 不引入第二套收藏/星标字段。
- 不绕过 UI 原型审批门禁。

## 需求与验收标准

| ID | 需求 | 验收标准 |
| --- | --- | --- |
| R1 | Project card 右键菜单 | 弹窗左侧任意 project card 右键后出现项目菜单；可切换到主空间、编辑项目元数据、星标/取消星标项目、归档项目。 |
| R2 | Space row/card 右键菜单 | 弹窗右侧所有 space row/card 右键后出现空间菜单；main 与 WorkTree 均支持；WorkTree 额外显示删除 WorkTree。 |
| R3 | 顶部三点菜单保留且分工清晰 | 顶部菜单仍可对当前工作区做编辑、星标、归档所有会话/当前空间等快捷动作；无需先切换的任意对象操作由弹窗右键承担。 |
| R4 | 星标文案统一 | 用户可见文案使用“星标”；不再新增数据字段；项目/空间星标均映射到 `pinned`。 |
| R5 | 项目排序保持既有规则 | 星标项目排前；同组按最近打开/更新时间倒序，再按项目名。 |
| R6 | 空间排序改为用户顺序 | 主空间固定第一；非主空间按持久化顺序显示；星标、最近打开、刷新同步不打乱顺序。 |
| R7 | 新空间默认追加 | 通过创建、刷新或发现产生的新非主空间出现在非主空间列表末尾。 |
| R8 | 排序可持久化 | 拖动后刷新页面、重新打开弹窗或重启服务，空间顺序保持。 |
| R9 | 保留 WorkTree 安全流程 | WorkTree archive/delete 的确认、dirty 提示、fallback 选择、session cleanup 行为与现有逻辑一致。 |
| R10 | 可访问性与交互反馈 | 右键菜单可关闭；拖动有把手、拖动态和落点反馈；缺失路径不能切换但仍可执行可用的元数据/归档类操作。 |

## 用户已确认的产品决策

- `project card` 与所有 `space row/card` 都需要支持右键菜单。
- Sidebar 顶部三点菜单需要保留，但与弹窗右键菜单重新分工。
- 星标可直接复用现有 `pinned` 字段，UI 文案统一为“星标”。
- 空间排序规则为：主空间永远第一；其他空间按用户拖动顺序；静默新增/新加入空间默认放在后面。

## 需要在本审批中确认的设计取舍

1. 接受 HTML 原型中的交互方向：顶部三点菜单保留 + project/space card 全量右键 + 非主空间拖动排序。
2. 接受推荐的持久化方案：为空间记录新增可选 `sortOrder?: number` 字段，并增加项目级批量空间排序 API，作为用户拖动顺序的持久化来源。
