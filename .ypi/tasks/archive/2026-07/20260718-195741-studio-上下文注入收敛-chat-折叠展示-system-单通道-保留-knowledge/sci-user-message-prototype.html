可访问性

  • Tag 是纯文本装饰（span，非 button/link）；L0 不抢焦点。
  • title 提供完整 status 文案；信息不以颜色为唯一通道（始终有 Studio · status 文字）。
  • Copy / Edit 保持可键盘聚焦；:focus-visible 使用 --accent 描边。
  • 消息容器 :focus-within 时显示 hover actions，便于键盘用户。
  • 对比度：明暗主题均用现网 status token；实现后人工扫一眼 dark 模式 tag。
  • 不要用仅 icon 的 tag；不要把注入全文塞进 aria-label（过长且 L0 不展开）。

UI 验收点（实现员 / 检查员）

  • 脏消息气泡不再渲染 <ypi-studio-state> / knowledge 大段正文。
  • tag 文案严格 Studio · {status}；未知注入 → Studio · context。
  • 干净消息：无 meta-row、无额外间距回归。
  • Copy / Edit from here 使用 displayText；失败路径用全文。
  • tag 右对齐于 user 列；与 bubble 间距约 4px；max-width 与 bubble row 一致。
  • 窄屏长 status 不横向撑破布局（ellipsis / shrink）。
  • 明暗主题对比度可接受；hover actions 显隐与现网一致。
  • L0 点击 tag 无展开 / 无 modal。
