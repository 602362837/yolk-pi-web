已完成“项目切换体验优化：session 重置、列表加载态、空间信息整合到预览区”。实现覆盖 P1/P2/P3，代码检查通过，checker 结论为 PASS，无 blocking。

# Reusable knowledge
- **项目切换重置要用 space 语义而不是 cwd 字符串**：将重置触发从 `handleCwdChange` 提升到 `activeProjectContext` 变化，用 `projectId + spaceId` 做 `spaceContextMatchesSession`，可统一覆盖弹窗切换、右键切换、WorkTree fallback、注册项目、git clone 等路径，同时避免 URL restore 被误清。
- **切换列表加载态要区分“空间切换”与“后台刷新”**：`loadSessions` 增加 `sessionsSwitching` 与 token 竞态保护；只有 space key 变化时才 skeleton + 清空旧列表，后台 refresh 继续 `showLoading=false` 保持平滑，旧请求响应必须按 token 丢弃。
- **FileExplorer 迁宿主要整体搬迁状态**：把 `explorerOpen/explorerHeight/explorerKey/explorerRefreshDone/explorerResizing` 与 localStorage 读写一起从 `SessionSidebar` 迁到 `AppShell`；新 key 用 `pi-web-preview-explorer-*`，首值兼容旧 `pi-web-sidebar-explorer-height`。
- **预览区内文件浏览器的 cwd 来源**：使用 `activeCwd ?? activeProjectContext?.cwd`，可在切空间后自动重载文件树，并与 reset 后右侧面板重新打开的场景兼容。
- **保持 sidebar 顶部选择空间区域不动**：本次迁移对象是 sidebar 底部 FileExplorer，不是顶部项目/空间切换区；该边界在设计和实现中需要明确。
- **收尾标准**：至少跑 `npm run lint` 与 `node_modules/.bin/tsc --noEmit`；checker 可通过静态审查确认大部分逻辑，但 8 类切换路径、快切竞态、拖拽边界仍需手工回归。
