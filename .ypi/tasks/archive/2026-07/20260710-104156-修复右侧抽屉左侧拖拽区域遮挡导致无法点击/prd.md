# PRD

## 目标与价值

修复右侧 Preview / Studio / Trellis 抽屉在桌面端的点击阻塞，使抽屉内容全宽可操作，同时保持左缘调宽。

## 范围内

- 仅修复 `> 640px` 下右侧抽屉 resize handle 的尺寸/命中区域。
- 保持 handle 位于左缘、`col-resize` 指针、拖拽调宽、拖拽中视觉反馈和现有最小/最大宽度约束。
- 覆盖三种右侧抽屉 mode。

## 范围外

- 不改移动端抽屉、右侧抽屉布局、持久化宽度、内容组件或交互文案。
- 不改变 resize 的手势阈值、键盘交互或新增设置。

## 验收标准

1. 在桌面宽度，右侧抽屉左缘仅保留约 8px resize 命中带；左侧其余内容（从 x=5px 起）可点击、滚动和选择文本。
2. 左缘命中带可向左/右拖拽，宽度仍受 300px 与 65vw 限制且持久化不回归。
3. Preview、Studio、Trellis 三种 mode 均无透明覆盖层拦截内容。
4. <=640px 时 resize handle 仍不可见且不可交互。

## 未决问题

无产品决策阻塞；但 UI 原型门禁必须完成并经用户审批后才可实施。
