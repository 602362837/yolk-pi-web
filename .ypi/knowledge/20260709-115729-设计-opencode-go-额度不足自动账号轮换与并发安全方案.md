## Summary
为 opencode-go 多账号实现了默认关闭的自动账号轮换方案，并完成实现、测试、UI 与文档。核心可复用结论：1）未发现可靠公开 quota/balance 查询 API，v1 应采用纯被动 failover；2）仅对明确 quota/billing exhausted 与明确 Invalid/Missing API key 等 account_unusable 错误触发，普通 429/network/5xx 不切号；3）account_unusable 应持久禁用账号，并要求手动 Enable 后才能再次使用；4）并发下需进程级锁 + 锁内 active-changed guard + activate 前 double-check，避免 A→B 后另一个会话再 B→C 级联；5）disabled 账号不得参与候选也不得激活；6）UI 需明确全局 active key 副作用，并提供 Enable/Disable、策略说明与轻量 Chat 提示。

## Reusable knowledge
# Summary

为 `opencode-go` managed accounts 实现了默认关闭的自动 failover、账号 Enable/Disable、并发防级联、Settings/Models UI 与 Chat 提示，并通过 lint、tsc 与专项测试。

# Reusable knowledge

- `opencode-go` 当前未发现可靠公开的 quota/balance 查询 API，v1 采用**纯被动 failover**更稳妥。
- 自动切换只应在**明确** `quota/billing exhausted` 或 `Invalid/Missing API key` 等 `account_unusable` 错误时触发；普通 `429/rate limit`、network、5xx 不切号。
- `account_unusable` 适合**持久禁用**触发账号；disabled 账号不得参与候选，也不得被设为 active，需用户手动 Enable 后恢复。
- 并发安全需使用**进程级锁 + active-changed guard + activate 前 double-check**，避免两个 session 从 `A->B` 继续级联到 `B->C`。
- UI 文案应明确：切换会修改**全局 active key**，并提供 Enable/Disable、禁用原因、恢复路径和非敏感 Chat 提示。
