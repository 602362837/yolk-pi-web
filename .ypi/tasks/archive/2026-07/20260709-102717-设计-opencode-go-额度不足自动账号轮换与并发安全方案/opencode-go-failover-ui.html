OpenCode Go 自动账号切换 (Auto Failover)
当当前激活账号发生额度耗尽、欠费，或者 API Key 永久失效被标记不可用时，系统自动在后台锁内切换至下一个可用的托管账号，并在当前会话重新发起当前请求 turn。
自动轮换策略安全边界
  • 精准匹配：仅支持额度不足（Usage Limit Exceeded、Balance Out of Budget 等）或账号失效（Invalid/Missing API Key）的错误。
  • 非切换错误：普通的并发限流 (429 Rate Limit)、网络超时 (Timeout) 或服务器 502/503 错误不会切换账号，依然交由系统默认重试或报错输出。
  • 全局影响：账号切换是全局行为，切换后新建的并发请求及后续 turn 均会使用新激活账号。
  • 防级联控制：每一轮 turn 中每个会话最多允许重试 1 次且仅切号 1 次，防止由于全部 Key 不可用导致的无限轮刷。
