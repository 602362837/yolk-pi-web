# Summary

已完成并归档：YPI Studio 任务浮窗在等待审批时支持计划审批书预览。主任务 `awaiting_approval` 与改进项 `waiting_plan_approval` 在 session 浮窗显示只读入口，点击后 modal 按需读取 `plan-review.md`；缩短查看路径，不改变聊天批准与 grant 门禁。

# Reusable knowledge

1. **只读预览 vs 批准门禁**  
   浮窗/modal 只能 GET `plan-review.md` 与打开相对链接；禁止 modal 内批准/拒绝/PATCH/transition。真实批准仍在绑定聊天写 `approvalGrant`。

2. **入口门控**  
   - 主任务：仅 `status === 'awaiting_approval'` →「计划审批书」  
   - 改进项：每个 `status === 'waiting_plan_approval'` 独立 →「计划审批书 · IMP-xxx」  
   非审批态不渲染 action row；入口放状态元信息后的独立可换行 row，不占详情箭头，不改 360px 宽度。

3. **按需读取，不投影正文**  
   Widget projection 只带 status/ids；打开 modal 后再  
   `GET /api/studio/tasks/{taskKey}/files?cwd=&path=plan-review.md&mode=read[&improvementId=]`  
   不把 Markdown 塞进轻量 session-link / studio-task projection。

4. **组件拆分**  
   - `lib/ypi-studio-task-preview.ts`：共享 `resolveTaskRelativeHref`、API URL、open link（Panel 与 Modal 共用，避免双份路径逻辑）  
   - `YpiStudioPlanReviewModal`：fetch、AbortController、retryToken、stale guard、empty/TBD、error 脱敏、MarkdownBody、dialog focus/Escape/遮罩、移动近全屏  
   - `YpiStudioSessionWidget`：生成 targets + 挂载 modal  
   - `AppShell`：传入授权 `cwd` 与 `handleOpenFile`

5. **多改进项不串读**  
   每个 target 显式 `taskKey + improvementId(+displayId)`；请求与 HTML preview 始终带同一 `improvementId`；切换 target 必须 abort 旧请求并用 stale response guard。

6. **安全边界**  
   客户端校验仅 UX；服务端 task-local resolver 拒绝 scheme、绝对路径、`..`、目录、符号链接逃逸。HTML 用 `mode=preview` + CSP sandbox。

7. **实现顺序可复用**  
   helper 抽取 → modal → widget 入口 → docs → lint/tsc/test:studio-dag；回滚仅删入口/modal/helper 接线，无 schema 迁移。
