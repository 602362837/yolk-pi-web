# Summary

优化 YPI Studio 计划审批书预览与产物去重已完成。实现了 `plan-review.md` 标准审批 artifact、任务详情“计划审批书”Tab、受控 Markdown 相对链接、任务本地 HTML 预览 API、Artifacts Tab canonical 去重，并修复审批门禁漏洞。

# Reusable knowledge

- awaiting_approval 阶段的审批材料应结构化为 `plan-review.md`，不要依赖 Chat 纯文本路径提示。
- 架构师 planning 阶段应生成计划审批书，并用 Markdown 相对链接引用 `prd.md`、`design.md`、`implement.md`、`checks.md`、`ui.md` 或 HTML 原型。
- `MarkdownBody` 的链接增强应做成可选 `onLinkClick`，避免影响普通 Chat Markdown。
- 任务目录文件/HTML 预览必须以服务端安全解析为边界：拒绝 URL scheme、绝对路径、`..` 逃逸、目录目标和符号链接逃逸；HTML preview 需要 CSP sandbox。
- Artifacts Tab 去重应基于 canonical `fileName`，合并 key/fileName/progress/documents 来源，并使用稳定排序：`plan-review` -> required -> optional -> mapping -> documents。
- 审批门禁安全不变量：`awaiting_approval -> implementing` 只能依赖后续用户输入通过 `recordYpiStudioUserApproval()` 记录的 grant，不能从 assistant-controlled `reason` 合成 approvalGrant。
