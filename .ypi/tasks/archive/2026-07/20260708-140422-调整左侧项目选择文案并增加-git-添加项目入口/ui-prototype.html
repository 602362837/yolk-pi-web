交互与状态设计细节 (v2)

  • 普通下拉 (1) 和 空态 (2)：Choose project folder… 变更为 Add project folder…，统一使用“增加”动作语义。
  • 直接提供双输入框 (3)：
    • Local parent path: 用户在本地的开发父目录。支持直接手动编辑回填或点击右侧文件夹图标通过系统管理器选择。
    • Remote repository: 远程仓库地址。
  • 父目录选择 Busy 态 (4) / Error 态 (5)：
    • 当拉起系统目录管理器时，仅仅父目录选择按钮进入等待（展示 ⌛），Add project folder… 的文案也随之变更，且表单不关闭。
    • 如果父目录无效（例如权限拒绝），在 parent path 输入框下方展示独立小错误提示。
  • Clone 执行 Busy 态 (6)：
    • 点击 Clone and add 后，两个输入框、选择按钮、Submit 及 Cancel 按钮均进入 disabled 态。
    • 提交按钮文字变为 Cloning… 并展示 Spinner/加载标志，防止重复点击。
  • Clone 失败 Error 态 (7)：
    • 在表单下方（Cancel / Submit 按钮之上）显示醒目的红色整体错误提示。
    • 失败后不切换当前激活的项目/空间，允许用户修改输入并重试。
  • 取消/关闭状态 (Cancel)：点击 Cancel 或外部空白区时，清空 Git 表单的所有临时文本及错误状态。

UI 验收点说明：

1. 提交按钮文案必须是 Clone and add，而非简单的 Add 或 Clone。
2. 输入框、文件管理器选择按钮在 busy 期间必须禁用，确保并发安全。
3. 错误消息展示位置：字段特定错误位于对应输入下方，整体流程错误（如 clone 失败）置于提交动作上方。
