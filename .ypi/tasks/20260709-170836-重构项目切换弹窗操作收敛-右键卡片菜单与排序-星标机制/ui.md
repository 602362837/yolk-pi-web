## 信息架构

### 1. Sidebar 顶部入口

保留两类入口并重新分工：

- **项目空间切换按钮**：负责打开 `ProjectSpaceSwitchDialog`
- **顶部三点菜单**：保留，但只负责“当前工作区相关的快捷动作 / 全局动作”

建议保留的顶部三点菜单项：

- 编辑当前项目元数据…
- 编辑当前空间元数据…
- 星标当前项目 / 取消星标当前项目
- 星标当前空间 / 取消星标当前空间
- 归档所有会话
- 归档当前空间
- （如是 WorkTree）删除 WorkTree…

### 2. 弹窗中的浏览态操作

`ProjectSpaceSwitchDialog` 仍保持“左侧项目、右侧空间”的两栏结构，但操作收敛到卡片本身：

- **左侧 project card：全部支持右键菜单**
- **右侧所有 space row/card：全部支持右键菜单**，包括：
  - main space
  - worktree space

右键菜单承担“浏览态下对任意目标的就地操作”，不用先切换到该项目/空间再去顶部菜单操作。

## 星标机制

- 数据层直接复用现有 `pinned` 语义，不新增第二套“收藏/星标”字段。
- UI 文案统一为“星标”。
- 展示方式：
  - project card：标题前或角标显示 `★`
  - space row：在名称前或拖动把手旁显示 `★`
- 星标的作用：
  - **项目列表**：继续影响排序
  - **空间列表**：主要承担“标记/快速识别”，不打乱用户手动排序

## 排序与拖动

### 项目列表

当前理解保持为：

- 星标项目优先
- 同组内按最近打开时间
- 再按名称

本轮先不扩展到项目拖动排序。

### 空间列表

用户已确认的规则：

1. **主空间永远第一**
2. **其他空间按用户拖动顺序排列**
3. **静默新增 / 新打开的空间默认追加到底部**
4. **星标不应改变上述主规则**

因此空间列表需要新增明显的拖动 affordance：

- main 空间无拖动把手
- 非主空间显示拖动把手（如 `⋮⋮`）
- 拖动中有占位、投放高亮或虚线框
- 新增项默认落在“非主空间列表”的最末尾

## 关键状态

| 场景 | 预期表现 |
| --- | --- |
| 右键 project card | 打开项目级上下文菜单 |
| 右键 main space | 打开空间级菜单，但不出现 WorkTree 专属动作 |
| 右键 worktree space | 出现 WorkTree 专属动作（如删除 WorkTree） |
| 星标项目 | 列表中显示 `★`，并继续影响项目排序 |
| 星标空间 | 显示 `★`，但不打乱用户拖动顺序 |
| 拖动非主空间 | 出现拖动态与落点反馈 |
| 新增/新打开空间 | 默认静默追加到底部 |
| 顶部三点菜单 | 仅处理当前工作区相关动作，不替代浏览态右键菜单 |
