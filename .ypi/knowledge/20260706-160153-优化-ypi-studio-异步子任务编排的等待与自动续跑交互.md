## Reusable knowledge
# Summary

完成 YPI Studio 异步/并行子任务编排优化：主 Chat 在等待并行 subtasks 时保持可见工作态，子任务 terminal 后自动唤醒同一主 session 续跑既有 Studio 状态机，直到继续派发、进入 checking、完成或请求用户处理。

# Reusable knowledge

- 不用独立后台 orchestrator 取代 Studio 状态机；低破坏方案是 **main Chat continuation loop**。
- 用户批准后才可进入 `implementing`；不得绕过 server-recorded `approvalGrant`。
- 派发 async implementer/checker 后，主 Chat 应投影为 `waiting_for_studio_children`，即使 `agentRunning=false` 也不能显示 stopped/idle。
- child run terminal 后注册 parent-session continuation，幂等唤醒同一主会话；由主会话执行 `collect -> update subtask -> implementation_next -> claim/dispatch -> checking/completed`。
- UI 通过 `sessionRuntime` + compact timeline 显示 `pending/ready/queued/running/waiting_for_user/done/failed/blocked/skipped`；Chat/Widget/Panel 文案使用“后台仍在工作”“正在等待并行子任务”“需要你处理”。
- 失败、`waiting_for_user`、blocked 等应进入 needs-user/attention，不继续静默派发依赖。
- 测试应覆盖 DAG readiness、maxConcurrency、waiting/needs_user projection、active child run counting、terminal continuation 幂等、approval gate。
