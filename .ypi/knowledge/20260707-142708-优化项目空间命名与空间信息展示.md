# Summary

优化项目空间命名与空间信息展示已完成：主空间默认展示从 `Main` 改为 `主空间`；左上角/侧边栏顶部空间摘要展示空间名、普通 Git 分支或 WorkTree 分支/基准；WorkTree badge/dropdown tooltip 补充基准信息；新增 WorkTree 可选 `baseRef` 元数据并在创建链路持久化、刷新同步时保留；相关模块文档已同步。

# Reusable knowledge

- 主空间中文化应只改 UI fallback，不要修改 `spaceId: "main"`、session JSONL header、Project Registry id 或历史链接。
- `displayName` 是用户自定义显示名，应继续优先；历史/手动 `displayName: "Main"` 不应被强制覆盖，除非另有迁移需求。
- SessionSidebar 顶部空间信息建议用集中 helper 格式化：
  - 普通 Git 空间：`空间：<name> · 分支：<branch>`；无分支时显示未检测到 Git 分支。
  - WorkTree 空间：`空间：<name> · WorkTree：<branch|未知分支> · 基准：<base|未知>`。
  - 缺失来源/基准时不要猜测为 `main`，应明确未知或使用已保存 metadata 兜底。
- WorkTree 来源分支需要创建时保存：在 Project Registry/Git metadata 中使用可选 `baseRef`；新建 WorkTree 写入 `result.baseRef`，同步/刷新时保留已有 `baseRef`，旧/外部 WorkTree 可 fallback 到 `mainWorktreeBranch` 或未知。
- UI tooltip/detail 可承载完整路径、分支、基准、missing 状态，列表行保持紧凑，避免因额外信息撑高。
- 回归风险：不要用 session 扫描替代 Project Registry；不要重写 session header；WorkTree refresh 不能丢已保存 `baseRef`。
- 验证命令：`npm run lint`、`node_modules/.bin/tsc --noEmit`；建议补充 dev UI 手工抽查主空间、WorkTree、missing path、长分支 tooltip。
