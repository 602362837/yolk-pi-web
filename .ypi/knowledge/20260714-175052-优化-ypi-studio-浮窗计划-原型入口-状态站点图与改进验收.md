## 结论
- 计划审批书/HTML 原型入口由完整 artifact registry 驱动常驻，不依赖 `awaiting_approval` / `waiting_plan_approval` 才生成；审批后保留并变色（已批准/已确认）。
- 「入口过了截断会消失」主因是状态门控 entry 生成，不是内容截断。
- 改进计划在任务详情与主计划审批书同级只读预览，显式 `taskKey + improvementId`；改计划只在 Chat。
- 浮窗八站：Brief → Design → Implement → Checks → Review → User Acceptance → Completed → Archived；运行证据优先，规划 md 不误标后段。
- 当前/待确认节点光圈 + 出站连接线流光；completed/archived 静态；尊重 reduced-motion。
- 改进结果验收：仅 `waiting_user_acceptance` → 确认 Dialog → 既有 `transition_improvement → accepted`；主任务不自动 completed。
- widget projection 有界：文件名/scope/审批态/动作目标，不带正文；预览 GET-only，不写 approval grant。
