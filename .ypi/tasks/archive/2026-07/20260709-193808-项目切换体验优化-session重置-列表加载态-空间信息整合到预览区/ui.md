# UI 设计交付门禁汇报：已重做 HTML 原型（v2）

为了解决上一轮由于理解偏差导致 P3 设计被推翻的问题，UI 设计员已重新设计并交付了符合最新 PRD 要求的 HTML 原型 v2。

## 变更说明

| 维度 | 上一轮作废原型 (v1) | 本轮已交付新原型 (v2) |
| --- | --- | --- |
| **侧边栏顶部** | 错误地移走了「空间选择与 WT 状态区域」 | **保持原样**：切换按钮 `⌘`、项目名/副标题、WT badge、Workspace action 三点菜单仍然都在 |
| **侧边栏底部** | 仍残留旧文件浏览器（未干净迁移） | **干净移除**：FileExplorer 完全从侧边栏底部移除，仅留会话列表与 Trellis 提示 |
| **预览区顶部** | 错误地放入了空间信息条 | **放入可折叠 FileExplorer**：右面板 (Files) 模式顶部可折叠展开文件浏览器，下方为 TabBar 与文件内容预览 |
| **折叠状态与高度拖拽** | 无 | 支持折叠标题行交互演示与底部高度拖拽条的视觉标记 |

## 新原型覆盖场景交互

1. **暗色与明色主题**：支持通过顶部 prototype-controls 切换。
2. **侧边栏顶部**：显示完整的项目选择按钮与 Workspace actions，演示了长项目名省略及 Hover Tooltip。
3. **Session 列表加载态**：演示了 P2 要求的 Skeleton 骨架屏占位与禁用状态。
4. **预览面板 (files) 顶部 FileExplorer**：
   - 展开态（默认）：显示标题「项目空间信息」、刷新按钮、折叠箭头，以及 FileExplorer 的目录树；
   - 折叠态：点击右侧向上折叠箭头，文件浏览器收起为一行标题栏，高度塌缩，下方预览内容区高度撑满。
5. **Studio/Trellis 模式**：点击右面板 Tab 切换到 Studio 或 Trellis，此时顶部的 FileExplorer 区块自动不显示，符合 prd 约定。
