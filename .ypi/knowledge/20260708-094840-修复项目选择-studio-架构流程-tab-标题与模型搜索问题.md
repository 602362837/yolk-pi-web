# Summary

完成“修复项目选择、Studio 架构流程、Tab 标题与模型搜索问题”并沉淀默认模板刷新补丁。主要覆盖：项目文件夹/path 添加语义、Studio UI HTML 原型审批门禁、浏览器 Tab 标题项目上下文匹配、模型 provider display name 搜索，以及后续补充的默认成员/流程显式拉取更新能力。

# Reusable knowledge

- 项目下拉中的 `Choose project folder…` 与 `Add project path…` 都应调用 `POST /api/projects`，并以返回的 `project.spaces.main` 作为选中目标。即使 `created=false` 代表项目已存在，也应刷新/复用返回记录并选中它；不要把“添加项目”入口退化为只提示或只能从列表切换。
- Studio UI 原型门禁要多层落点：`lib/ypi-studio-agents.ts` 的 architect/checker 默认模板、`lib/ypi-studio-workflows.ts` 的 feature/bugfix/ui-change 状态说明、`lib/ypi-studio-extension.ts` runtime prompt、以及项目本地 `.ypi/agents` / `.ypi/workflows`。触发条件包括页面变更、前端功能新增、已有交互变化、审批/确认体验变化、用户可见信息结构变化；必须指派 UI designer 输出 HTML 原型并经用户审批。
- 仅修改内置模板不足以更新已有项目 `.ypi`：Studio 面板/API 需要显式 `overwriteDefaults` / “拉取最新默认配置”，覆盖同名默认成员/流程文件，但保留自定义命名文件。
- Tab 标题不要只比较 cwd 字符串；应优先 `projectId/spaceId`，再用轻量路径归一化比较，最后 fallback 到 cwd/Git title。默认 metadata title 应复用统一 fallback 常量。
- 模型搜索若要支持提供商展示名，应在 `/api/models` 的 `modelList` 增加可选 `providerDisplayName`，前端 ChatInput/Settings 的 `ModelSelectOption.keywords` 纳入 provider id 和 display name，同时保持 `provider/modelId` value 与 set_model payload 不变。
