# Summary

YPI Studio 子代理卡死、主 session Stop 无效与 `ERR_STRING_TOO_LONG` 崩溃问题已完成修复。核心根因是 `ypi_studio_subagent` 子进程 stdout/stderr 被无界累积，并在结束时 `Buffer.concat(...).toString()` 生成超大字符串。实现改为 bounded JSONL 流式解析，并补齐子代理取消链路与输出安全上限。

# Reusable knowledge

- 子进程 stdout/stderr 不应完整缓存后再转字符串；应使用 `StringDecoder` 流式解析并保留 bounded tail。
- 对 Studio 子代理必须同时限制 stdout 总量、stderr 总量、stdout 单行、final output、live preview、transcript sidecar 与 transcript API 响应。
- 主 session `abort`/`destroy` 需要级联取消 Studio child run；通过独立 runtime registry 避免 `rpc-manager` 与 extension 循环依赖。
- Stop HTTP 路径不能无限等待底层 SDK abort；应使用短超时返回，同时让底层 cleanup 异步收尾。
- 子代理 idle timeout、max runtime、output-limit 应记录为 failed/cancelled + warnings，不自动推进任务状态。
- UI 应明确展示 truncated、failed、cancelled，并提示重试、从当前阶段继续或标记 blocked/cancelled。
