# Summary

完成 Web Terminal SSH 凭证管理、多级跳板机与代理连接功能。实现范围包括 SSH profile/config/credential 类型、credential vault、dedicated profile CRUD/test API、known_hosts 管理、OpenSSH runner、ProxyJump、SOCKS5/HTTP proxy helper、askpass、临时文件清理、terminal session `kind=local|ssh` 集成、Settings SSH 管理 UI 与 TerminalPanel SSH profile picker。最终 checker re-review verdict 为 **Pass**，验证通过：`npm run lint`、`node_modules/.bin/tsc --noEmit`、`npm run test:terminal-ssh`。

# Reusable knowledge

- `pi-web.json` 只能保存非 secret profile/config；私钥、password、passphrase、proxy password 必须进入独立 vault，并且 credential API 只返回脱敏 summary。
- profile create/update API 与 pi-web config normalize/validation 都需要递归拒绝 secret 字段，避免通过 nested target/jump/proxy 绕过边界。
- redacted SSH launch plan 不应暴露 secret，也不应暴露真实 session tempDir；使用占位符更安全。
- built-in SOCKS5/HTTP proxy helper 应通过临时 context 文件传递代理认证，避免把 proxy password 放入命令行。
- known_hosts 独立于 `pi-web.json`；`ssh-keyscan` 结果只能作为 advisory，UI/文档需提示用户通过可信渠道校验 fingerprint。
- custom ProxyCommand 是本机命令执行能力，应要求全局开关与 profile 级风险确认，且拒绝 secret 占位符/控制字符。
- Settings → Terminal 是 SSH credential/profile/known_hosts 的主要录入口；TerminalPanel 只负责从已有 profile 打开 SSH tab。
- 预览/排查时要确认 dev server 的 cwd。此次 30142 先跑在主仓库而非 worktree，导致用户看不到新 UI；重启到 worktree 后恢复。
