# plan review

本任务针对**项目切换弹窗中项目支持手动拖拽排序，且星标解耦为纯视觉标记，不再干扰排序**的需求开展。该计划书由 UI 设计员根据 intake briefing 进行梳理，并编写了交互式 UI 原型，供主会话与用户审阅。

## 关联产物
- [UI 设计说明](ui.md)
- [交互式 HTML 原型](project-switch-drag-prototype.html) (支持切换主题、Drag-and-Drop、星标解耦点击无跳动演示)

## 方案简述
1. **项目手动排序**：
   - 数据层新增项目级 `sortOrder?: number` 属性，并新增 `PATCH /api/projects` 批量更新排序接口。
   - `ProjectSpaceSwitchDialog` 左栏增加项目拖拽把手 `⋮⋮` 与对应的拖拽状态高亮。
2. **星标语义解耦与零跳动**：
   - 项目与空间的星标 (`pinned` 属性) 退居为纯视觉标记，在展示排序中不再起主导作用。
   - 用户点击星标后，列表只改变星标颜色 (★/☆)，卡片物理位置保持稳定，彻底消除“点击星标列表立刻跳动”的交互抖动痛点。
3. **搜索态防护**：
   - 搜索状态下若允许局部拖动重排会导致全局顺序错乱。原型中规定：搜索关键词非空时，拖拽把手展示为禁用态并提供气泡/横条提示：“搜索状态下禁用拖动排序，请清空搜索后重新排序”。
