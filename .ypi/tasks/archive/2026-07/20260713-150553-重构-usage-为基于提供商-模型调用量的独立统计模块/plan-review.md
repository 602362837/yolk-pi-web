# 计划审批书：新增独立 Provider / Model 用量限制模块

## 用户确认后的方向

用户确认：保留原先的 Usage，不删除、不替换；在此基础上额外增加一套独立的“用量/限制”统计模块。新模块直接记录 LLM 调用完成后的 provider/model 用量，作为未来限额、配额和计费的基础。

## 人话版方案

1. 原 Usage 继续工作，保证现有功能不受影响。
2. 每次 LLM 调用完成时，旁路写入一条独立用量记录：输入、输出、缓存写入、缓存读取，以及 provider/model。
3. 新记录不依赖 session，因此 session 删除或归档不会影响新模块。
4. 新模块先只做记录和展示，不改变模型调用行为，也不阻断聊天。
5. 先新旧并行运行和验证；确认数据稳定后，再逐步增加额度/限制规则。
6. 如果某个调用入口无法通过公开稳定的 SDK 回调获取 usage，会显示统计缺口，不使用私有 monkey patch 硬拦截。

## 关键安全边界

- 不直接改底层 HTTP 请求，不修改 API Key 或模型请求参数。
- 不保存 prompt、回复内容、文件路径、凭据或账户信息。
- 新用量记录写失败不得影响正常 LLM 调用。
- 原 Usage 永远保留为兼容/回退路径。
