# UI

## UI prototype gate：已触发，交付确认

本任务改变现有点击交互、增加 Detail 控件、改变收纳/抽屉可见性、加入动画以及设计“站点连线式”进度拓扑图，由 **UI 设计员** 产出了基于当前 `YpiStudioSessionWidget` 风格的独立单文件 HTML 原型。

交付形式：在本任务目录更新并覆盖了 [ui-prototype.html](./ui-prototype.html) 原型文件，该文件为纯 HTML + CSS + Vanilla JS 结构，支持离线或以任意浏览器直接打开进行动态交互、切换和演示。

## 原型核心变化点与特性说明

1. **单文件零依赖**：旧原型因采用外部 Tailwind CSS CDN 以及某些外部依赖，在沙箱隔离或离线环境下可能无法正确渲染。本版原型重新以**纯内联 CSS 变量与原生样式**重构，同时保持高水平的像素还原度与多端适配，双主题（深色模式/浅色模式）切换极其流畅。
2. **任务球收纳防消失保护**：
   - 展现态分为 `expanded | collapsed`。收纳为悬浮球后，不再因为 Studio Drawer 打开或命中聚焦而被隐藏。
   - 在原型中，可通过左侧面板模拟 `drawerOpen && focusedBoundTask`，可以清晰地观察到：此时悬浮球/面板仅呈现轻微淡出（避让），不渲染为 `null`，依然能够顺畅点击交互。
3. **Detail 按钮交互分责**：
   - 每张任务卡片右上角新增半透明圆形毛玻璃 `Detail` 按钮（带右指箭头 `svg`），并且设置了独立的 pointer click 捕获。
   - 点击卡片其他空白或文本内容只触发“阅读态高亮”，不会误触弹出右侧 Drawer；只有明确点击 `Detail` 按钮时，才会弹出 Studio Drawer，有效解决了大卡片误触和滚动冲突的问题。
4. **站点连线进度（Workflow Timeline）**：
   - 放弃了纯文本 progress，采用扁平的“路线地图”表达：卡片内渲染一条清晰横向拓扑进度线，依次展示 `Brief -> Design -> Implement -> Checks -> Review`。
   - 当前正在进行的状态会以状态色（Running 蓝色呼吸 / Waiting 橙色强调）在相应节点处亮起并标注，已完成的节点呈现绿色成功状态，一眼即可识别出当前 Workflow 所在生命周期节点。
5. **多任务堆叠与设备响应**：
   - 提供了多任务（3张卡）在展开面板/移动底栏的堆叠渲染模拟，各个 Detail 按钮不重合，拖拽 Header 与卡片内滚动边界分明。
   - 提供了桌面端和移动端半屏 Bottom Sheet 的切换效果。
6. **动效隔离与 Reduced Motion 降级**：
   - 状态呼吸 Halo 与脉冲 Pulse 作用在球体表现层，拖动 inline transform 作用在壳体外层，完全避免了位移抖动。
   - 提供了 prefers-reduced-motion 无障碍检测与降级按钮，开启后自动移除一切过渡与关键帧动效，符合现代体验标准。
