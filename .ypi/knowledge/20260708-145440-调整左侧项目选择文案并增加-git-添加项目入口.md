# Summary

完成左侧项目选择改造：普通目录入口文案改为 `Add project folder…`，并新增 `Add project from Git…`。Git 流程使用 `Local parent path` + `Remote repository` 两输入，支持 `https://...` 与 `git@host:user/repo.git`，clone 成功后自动注册项目并选中 main space。

# Reusable knowledge

- 项目选择的普通目录添加与手动路径添加，仍应统一走 `/api/projects` 注册语义，并以返回的 `project.spaces.main` 作为最终选中目标。
- Git 添加项目不应把父目录注册为项目；`Local parent path` 只是执行 `git clone` 的父目录，真正注册的是 clone 产出的目标项目目录。
- `Add project from Git…` 表单应与 `Add project path…` 表单互斥；取消、Escape、外部关闭下拉、以及直接切换项目空间时，都应清理 Git 临时状态，避免残留输入和错误。
- `/api/projects/select-directory` 适合兼容扩展 `purpose=project|git-parent`，但服务端只能把 enum 映射到固定选择器提示文案，不能接受任意客户端 prompt。
- `POST /api/projects/git-clone` 应使用非 shell 的 `execFile` 参数数组执行 git，支持 `https://` 和 `git@...` remote，校验父目录、git 可用性、目标目录已存在等错误，并在 clone 成功但注册失败时返回 `clonedPath` 供用户手动恢复。
