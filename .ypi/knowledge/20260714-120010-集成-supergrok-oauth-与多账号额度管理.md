# Summary

SuperGrok OAuth 已集成到 Web：使用 `pi-grok-cli@0.4.1` 提供 OAuth、Grok CLI provider 和模型；支持多账号安全保存、激活、刷新、会话绑定和额度展示。额度来自 `/billing` 与 `/billing?format=credits`，支持月度额度、可选周额度、缓存和失败降级。

# Reusable knowledge

- `pi-grok-cli` 的入口是 TypeScript 源码，内部大量使用 `.js` ESM 相对引用。Next/Turbopack 静态导入会报 `Unknown module type` 或无法解析 `./provider/register.js`；在 Next 服务端应通过 `jiti` 异步加载，并将 `jiti`、`pi-grok-cli` 放入 `serverExternalPackages`，避免同步 require 与 Pi loader 的 ESM race。
- Pi provider 必须通过统一 bootstrap 注入所有 `createAgentSessionServices`、`DefaultResourceLoader`、Models/Auth 路径；否则 registry refresh 会丢失动态 provider。
- OAuth 多账号不能只切换全局 `auth.json`：保存 opaque storage id 和独立 secret，active 仅影响新会话；已有 session 必须持久化 account pin，refresh 按账号 single-flight 并原子回写。
- Grok quota 是非公开 CLI 后端接口，严格解析 `monthlyLimit`、`used`、`billingPeriodEnd` 与可选 weekly 字段；推荐 60 秒 fresh、24 小时 stale、401/403 refresh retry、`Cache-Control: no-store`，不返回原始 upstream payload 或 token。
- Vision/Imagine 可能绕过主 inference header hook，需后续关注 per-call token override；billing schema 变化也应通过 parser fixtures 防护。
- 验证基线：`npm run lint`、`tsc --noEmit`、provider 40/40、accounts 70/70、quota 48/48、`git diff --check`。
