# 预览区拖拽、右侧抽屉与多任务任务球

## Summary

完成预览区垂直拖拽方向修正、右侧抽屉宽度调整与持久化、多任务 session 绑定感知，以及任务面板收纳为可拖动任务球。

## Reusable knowledge

- 预览区垂直拖拽高度计算应与用户拖动方向一致，并保持 explorer/preview 最小高度 clamp。
- 右侧抽屉宽度使用共享状态、localStorage 持久化和 viewport clamp；移动端保持全屏 drawer、隐藏 desktop resize handle。
- session link resolver 应返回当前 session 的全部明确绑定 task，避免多任务时因 `ambiguous` 将所有任务置空；未绑定 transcript 提及不得进入 widget。
- API 兼容保留旧 `task` 字段，同时提供 `tasks[]`、`primaryTaskKey` 与 warnings/diagnostics；自动继续只作用于 primary/current 已绑定 implementing task。
- widget 采用多任务卡片堆叠；collapsed 任务球和 expanded 面板均可拖动，位置必须 clamp 在可视安全区并在 resize/刷新后恢复。
- overlay、轮询、聚焦和任务入口按 taskKey/taskId 分发，不应只绑定 primary task。
