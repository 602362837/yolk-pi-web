# Summary

完成 Project Registry 与 Project/Space 层级改造：Project Registry 成为左侧项目树唯一顶层数据源，Project 下包含 `main` 与 Git worktree spaces，Session 作为 space 下历史记录，通过 session header 的 `projectId/spaceId` 关联。任务完成 6/6 个实现子任务，Checker Verdict 为 Pass，`npm run lint` 与 `node_modules/.bin/tsc --noEmit` 均通过。

# Reusable knowledge

- 项目列表不要从 sessions 聚合，应使用独立 `pi-web-projects.json`；session 只用于 space 历史。
- 路径比较必须使用 canonical `pathKey`/realpath，UI display path 不能作为去重或安全校验依据，可避免 `/Users` 与 `/Volumes` alias 重复。
- Session header 是 project/space link 真源，`pi-web-session-index.json` 只作性能索引；旧 session 缺字段应可读但不要自动回写。
- 创建 project-scoped draft/new session 时，写入 header 后若强制 `_rewriteFile()`，需同步 SessionManager flushed 状态；否则首 prompt 可能持久化异常。
- Agent prompt preflight 失败不能静默，必须 emit `agent_error` 与 `agent_end`，避免 UI 永久停在 Waiting for model。
- 单 session detail API 不应调用 `listAllSessions()`；详情页只解析目标 session，列表只返回摘要。
- `listAllSessions()` 默认不应计算 Git/worktree 元数据；需要时通过 opt-in 参数懒加载或独立接口缓存。
- Project-space sidebar 下归档 session 应显示 archived count；未关联旧会话可按产品要求隐藏，不参与空状态判断。
