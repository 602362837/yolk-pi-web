### 覆盖与状态说明
1. **缺价模型列表与筛选**：主表格列出不同状态的模型，顶栏支持 `全部 / 缺少价格 / 已配置 / 免费` 筛选。
2. **手工编辑**：侧边 Drawer 支持手动修改三项价格，开启“显式免费模型”开关后单价置为 0，且输入框禁用淡化；带非负十进制校验。
3. **智能填写与建议**：抓取状态包含加载动画，展示精确匹配（高置信度）和冲突（低置信度警告并默认不勾选）及未找到匹配等状态。
4. **差异确认**：提交保存时展示 Diff 差异对比，注明对历史数据无追溯性重算。
5. **并发冲突与失败模拟**：右上角带有快捷调试器可快捷触发 `409 并发冲突` 浮窗，以及 `AI/网络抓取失败` 的超时状态界面。
6. **Usage 精确值 + M 换算展示**：切换至“使用量与账本”标签页，展示 `精确整数 (12,584,291 tokens) + M 派生 (12.584291 M)` 并列排版，去除冗余 cacheWrite 字段。

## 页面建议

入口：Settings 左侧导航新增“模型价格”，与 Usage 相邻；不要塞进现有通用 Models provider 编辑详情。

桌面布局：

- 顶部：搜索、`全部 / 缺少价格 / 已配置 / 免费`筛选、智能填写按钮。
- 主表：Provider / Model、价格状态、Input、Output、Cache Read、来源、更新时间、编辑。
- 主值使用 USD / 1M tokens；缺价显示明确的“未配置”，不能把 0 伪装为免费。
- 行编辑抽屉：三个价格输入、免费模型开关、来源备注、保存/取消。
- 智能填写弹窗：识别目标 -> 抓取中 -> 候选差异 -> 用户逐项勾选 -> 二次确认 -> 保存结果。
- 建议字段旁展示置信度与引用链接；冲突/低置信度为警告色且默认不选。
- 保存确认明确写入 `models.json`，说明“只影响后续调用，历史费用不重算”。

移动端：

- 表格转为紧凑行列表，点击进入近全屏底部 sheet。
- 固定底部操作区不得遮挡字段错误与来源。

## Usage 展示建议

- 精确值为主：`1,234,567 tokens`。
- M 值为次：`1.234567 M`，视觉弱化但不只放 tooltip。
- 紧凑顶栏可用 `1.235 M`，其 tooltip 必须显示 `1,234,567 tokens`。
- 删除 Cache Write 行/列后，Cache Read 单独展示；不要保留歧义的 `R/W` 标题。

## 必须覆盖的原型状态

1. 缺价模型列表与搜索/筛选。
2. 手工编辑：正常、字段错误、合法免费模型。
3. 智能填写：加载、可靠候选、来源冲突、低置信度、无结果、网络/AI 失败。
4. 差异确认、保存中、成功、409 并发冲突。
5. 桌面与窄屏。
6. Usage 移除 Cache Write 后的 Token 拆分与 exact + M 展示。
