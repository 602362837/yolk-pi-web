## Reusable knowledge
# Summary

修复 Chat 顶部与统计中的 Studio 子任务计费汇总显示错误，并补充顶部 hover 浮窗展示本会话与子会话费用构成。

# Reusable knowledge

- 顶部计费不要硬编码 `+child`，也不要用 `childCount > 0` 代表存在子任务计费；应按真实 child token/cost totals 判断。
- 展示口径建议固定为三类：
  - `parent`：显示 parent rollup；有真实子任务计费时用 `incl. Studio` 标记。
  - `standalone`：只显示自身费用。
  - `studio_child`：compact 只显示当前 child 自身费用；hover/tooltip 可补 `parent rollup`。
- 若 parent/child tab 切换后顶部展示依赖口径字段，刷新 key 必须纳入 `selectedSessionKind`、`selectedSessionTotals`、`parentRollupTotals`、child token totals 等驱动字段，否则总额相同时可能不刷新。
- Usage rollup API 应优先走 additive 扩展：保留旧字段语义不变，新增 `selectedSessionTotals`、`parentRollupTotals`、`selectedSessionKind` 供 UI 消费。
- 回归验证建议用 fixture 覆盖 `global totals`、`bySession`、`byParentSession`、`session_rollup(parent/child)`、`includeArchived`，确保 parent totals = own + child。
