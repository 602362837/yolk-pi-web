## Reusable knowledge
# Summary

Usage 重构已完成并由用户人工验收通过。旧 Session Usage 保留；新增独立、按 provider/model 聚合的 LLM 用量账本，记录输入、输出、缓存写入和缓存读取。账本默认启用，Chat 调用可写入，且不依赖 session 删除或归档。

# Reusable knowledge

- 账本数据应在 LLM completion 终态旁路记录，不能依赖 session JSONL；写入失败不得阻断聊天。
- 默认配置必须启用 ledger；`legacy` 仅作为显式回退模式，否则新账本会出现空数据。
- 事件写入需要稳定幂等 ID、隐私白名单和原子持久化；不保存 prompt、回复、凭据或绝对路径。
- 保留旧 Usage 作为兼容/回退，新增视图必须支持双向切换。
- 用量 UI 默认显示完整数字，并提供 hover 细节；Provider/Model 明细展示缓存命中率。
- 每日模型占比使用堆叠柱状图，默认按使用量，可切换费用；费用缺失不能影响使用量统计。
- 图表 tooltip 应是独立浮窗卡片，包含日期、模型、区域值、占比和当日汇总，具备背景、边框、圆角、阴影、高层级和窄屏防溢出定位。
- 新增 UI 变更必须同步 HTML 原型、审批记录和键盘/窄屏验证。
