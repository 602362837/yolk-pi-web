# Summary

已完成并归档：侧栏 session 列表热路径移除归档读取与展示。

目标：归档浏览极低频，不应拖慢默认 session 列表。实现选择硬切断侧栏热路径上的 archive 读/展示，而不是复杂 lazy-load。

## 结果

- `GET /api/projects/:projectId/spaces/:spaceId/sessions`：不再 `scanArchivedCwds()`，响应为 `{ sessions, legacyUnassigned, studioChildrenByParentSessionId }`（无 `archivedCounts`）。
- `GET /api/sessions`：响应收敛为 `{ sessions }`，无 archive cwd/count 字段。
- `SessionSidebar`：删除 archived state/loader/展开区块/`ArchivedSessionItem`/unarchive；空态与 archive-all 只看 active；归档写动作保留并只 `loadSessions(false)`。
- 文档：`docs/modules/api.md`、`frontend.md`、`docs/architecture/overview.md` 同步 active/explicit archive 分离。
- 检查员 **Pass**；lint/tsc 通过。

# Reusable knowledge

1. **低频能力不要放在列表热路径**：archive count/scan 即使有 TTL cache，仍会在每次 active list 时触发 `sessions-archive` 目录 I/O；侧栏几乎不用时，应直接从 hot path 删除，而不是只隐藏 UI。
2. **保留存储与显式 API，切断默认投影**：`scanArchivedCwds` / `listArchivedSessionsForCwd` / archive|unarchive|archive-all|archived routes / Usage `includeArchived` 可保留；默认 Sidebar 与 list routes 只服务 active。
3. **不要用空兼容字段糊弄**：删除 `archivedCounts`/`archivedCwds` 时不保留空壳；仓内确认消费者（如 `ypic` 只读 `sessions`）后再收敛响应。
4. **归档写与读分离**：single/batch/archive-all 成功后只刷新 active；不要为了刷新再拉 archived list。
5. **性能边界要诚实**：去掉 archive 热路径只保证少做一类 I/O；若 active 仍慢，主因多半是全局 `listAllSessions()` + filter，需另开 space 定向扫描/索引任务。
6. **UI 门禁**：删除用户可见信息结构（已归档区块）需 HTML 原型审批；实现与原型对齐重点是信息结构删除与计数口径，文案可有 low 级差异。
