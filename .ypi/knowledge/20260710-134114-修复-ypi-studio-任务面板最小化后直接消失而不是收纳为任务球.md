# 修复 YPI Studio 任务面板收纳、详情与多任务感知

## Summary

修复任务面板最小化后消失问题，并完成多任务悬浮 widget、任务球、Detail-only 详情入口、五站 workflow 路线和 artifact 标注。

## Reusable knowledge

- `collapsed` 与 `hidden` 必须分离；有 bound task 时始终渲染可交互任务球，drawer/focus 不能 `return null` 或隐藏入口。
- 多任务 API/widget 只纳入明确绑定当前 session 的 task；transcript-only 或未绑定 task 不得占位或替换旧任务。
- 卡片本体保持只读；每卡独立 Detail button 才能打开详情。面板拖动仅限 header，球使用 drag threshold 区分点击展开与拖动。
- workflow rail 应优先使用显式 workflow/runtime step；planning 阶段存在 `checks.md`/`review.md` 不代表运行时 Checks/Review 已完成。
- artifact 标注必须使用任务详情的完整 artifact registry，规范化 key/fileName 并去重，不能只使用当前状态的 required artifacts。
- 位置初始化、ResizeObserver、viewport clamp 与 localStorage 要覆盖条件挂载和 collapsed 切换；移动端保持 bottom sheet。
