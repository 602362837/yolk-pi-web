# Summary

本任务规范了 YPI Studio 的 UI 设计员默认模板，要求 UI 设计员必须产出 HTML 格式原型图，并在交付实现前上报主会话 / 用户审阅；确认前不得进入实现。实现更新了 `lib/ypi-studio-agents.ts` 默认模板、当前项目 `.ypi/agents/ui-designer.md`，并在 `docs/modules/library.md` 记录行为。

# Reusable knowledge

- YPI Studio 默认成员模板源头在 `lib/ypi-studio-agents.ts` 的 `DEFAULT_YPI_STUDIO_AGENTS`。
- UI 设计员模板应明确：HTML 原型是 UI 设计交付门禁；Markdown、表格或图示只能作为补充，不能替代 HTML 原型。
- UI 设计员输出格式建议包含 `HTML Prototype` 和 `Review Request`：前者要求自包含 HTML 原型，后者要求主会话 / 用户审阅并确认。
- 默认成员初始化/迁移必须避免覆盖用户自定义模板：只对精确匹配已知旧默认 hash 的文件执行更新，未知内容保持 `skipped`。
- 当默认模板变更时，应将上一版默认文件 hash 加入迁移白名单；本次把 `OLD_DEFAULT_AGENT_HASHES` 从单 hash 改为多 hash 集合，并保留 `ui-designer.md` 的 `d728c01f...` 与 `e8957ea...`。
- 推荐验证：`npm run lint`、`node_modules/.bin/tsc --noEmit`、搜索旧替代说法是否消失、搜索 HTML 原型与 Review Request 是否存在、检查 hash 白名单是否保留。
