## Reusable knowledge
# Summary

完成 YPI Studio Members tab 成员模型配置展示与快速修改入口：成员卡片和详情显示模型、thinking 与来源，点击“修改模型”打开 Settings 的 Studio section 并滚动/高亮对应成员配置行。支持默认成员和从 Members tab 跳转的自定义成员；Settings 常规入口保持原行为；未新增 API，未改变 runtime 模型解析优先级。验证 `npm run lint` 与 `node_modules/.bin/tsc --noEmit` 通过。

# Reusable knowledge

- 成员模型配置来源是 `~/.pi/agent/pi-web.json` 的 `studio.defaultPolicy` 与 `studio.members[memberId]`，不要写入 `.ypi/agents`。
- Members tab 展示层可复用 AppShell 已加载的 `/api/web-config`，按 member id 读取成员策略，缺失或 `unset` 时展示默认策略/回退链。
- 快捷修改链路：`YpiStudioPanel` 调用回调，`AppShell` 打开 `SettingsConfig` 并传 `initialSection="studio"`、`studioFocusMember`；Settings 行通过 ref scroll/highlight。
- 自定义成员建议只在 Members tab 跳转时显示临时配置行，保存写入 `studio.members[id]`；常规 Settings 入口保持默认成员配置。
- 成员卡如需内嵌“修改模型”按钮，应避免嵌套 button：卡片用非 button 容器 + `role="button"`/`tabIndex`，保留 Enter/Space 选择。
- `unset` 文案要避免递归/含糊的“使用默认策略”，应明确提示最终回退如“跟随主会话 → Pi 默认”。
